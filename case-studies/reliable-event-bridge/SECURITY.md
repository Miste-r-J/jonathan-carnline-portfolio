# Security and publication boundary

This repository contains only synthetic data and a domain-neutral reference implementation.

Before publishing changes, verify that commits contain none of the following:

- credentials, tokens, keys, cookies, or connection strings;
- personal account or customer identifiers;
- private IP addresses, hostnames, or filesystem paths;
- proprietary configuration, private research artifacts, datasets, or research values;
- production logs or screenshots; or
- copied source from the private implementation.

Security issues in this demonstration should be reported privately to the repository owner rather than through a public issue containing exploit details.
