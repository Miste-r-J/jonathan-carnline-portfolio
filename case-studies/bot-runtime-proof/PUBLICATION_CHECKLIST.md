# Publication Checklist

Before this went into the public portfolio, I checked for the things that should not be online.

## Excluded

- credentials, tokens, webhook URLs, passwords, API keys;
- account numbers, broker account aliases, funded-account identifiers;
- raw run logs, raw screenshots, raw fills, and account-level traces;
- model thresholds, strategy internals, and private research logic;
- large archives or dependency folders;
- local machine paths except where used in this checklist for audit context.

## Included

- engineering summaries written in my own voice;
- architecture notes;
- safe failure-mode examples;
- evidence maps;
- reviewable implementation patterns without production strategy details.

## Recruiter read

This package is meant to support roles in:

- production support;
- application support;
- operations technology;
- Python automation;
- C# / Windows integration;
- DevOps and reliability;
- logistics or real-time operations systems.
